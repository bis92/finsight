# PRD: finsight

> 계획의 단일 원천은 `plan.md`다. 이 문서는 plan.md에서 파생된 제품 요구사항 요약이며, 상충 시 plan.md가 우선한다.

## 목표

임의 CSV(은행 거래내역·카드 명세서)를 올리면 Claude가 컬럼을 자동 인식해 표준 스키마로 정규화하고, 소비 인사이트를 대시보드로 보여준다. 엑셀 수작업 없이 "내 돈이 어디로 갔고, 무엇을 바꿔야 하는지"를 즉시 파악하게 한다.

## 사용자

여러 카드·계좌 지출을 한눈에 정리하고 싶은 개인(B2C 가계부/소비분석). 은행/카드사 앱 분석이 계좌별로 파편화돼 통합 뷰가 없다고 느끼는 사람.

## 핵심 기능

1. **CSV 업로드 + 컬럼 자동매핑** — 인코딩 감지(UTF-8/EUC-KR) → 파싱 → LLM이 헤더+샘플 ≤20행으로 컬럼 매핑 → 사용자 매핑 확인 스텝.
2. **Free 대시보드(사실 조회)** — 이번 달 카테고리별 지출, 상위 가맹점, 총지출/수입(순지출), 기본 차트(파이·바). 상세는 SideView, 재분류는 Modal.
3. **Pro 인사이트(해석·코칭)** — 심화 LLM 인사이트/코칭 + 정기구독 후보 탐지 + 절감 제안. 페이월은 서버 게이팅(`profiles.plan`).
4. **게스트 데모** — 가입 없이 샘플 fixture로 대시보드를 읽기전용 체험 → "내 파일로 해보기" CTA로 가입 유도.
5. **결제 업그레이드** — Polar 구독 체크아웃, 웹훅으로 `profiles.plan` 갱신.

## 과금 모델 (ADR-006)

기능 **깊이**로 가른다(횟수 제한 아님 — 명세서는 월 1회라 횟수 제한이 무의미).

- **Free** = "내가 뭘 썼나"(집계, LLM 가벼움).
- **Pro** = "그래서 뭘 해야 하나"(LLM 추론, 무거움). Pro 앵커 2개: **심화 인사이트**, **정기구독 후보 탐지**.
- 단일 파일에 여러 달 거래가 있으면 merchant+amount+cadence로 반복 후보 탐지, 단일 월만 있으면 패턴 기반 "후보"로 표시. **확정 표현 금지.**

## MVP 제외 사항 (로드맵)

- 은행/카드 API 직접연동(오픈뱅킹·스크래핑)
- 다중파일 병합·중복감지 (MVP는 **단일 파일 업로드**. 단, 한 CSV에 여러 달 포함 가능)
- 다월(MoM) 비교/추이 UX, PDF 리포트
- 팀/멀티유저·예산설정, 일반 API rate limit/스케일링, 모바일 앱, i18n, 다크모드

## 성공 기준

- **Activation:** 신규 사용자가 샘플 데모 또는 자기 CSV로 3분 안에 첫 대시보드를 본다.
- **업로드 품질:** 대표 fixture 기준 파싱 성공률 95%↑, 필수 컬럼 자동매핑 성공률 80%↑. 실패 시 수동 매핑 폴백으로 업로드 전체가 막히지 않는다.
- **인사이트 가치:** Pro 화면은 "심화 인사이트"·"정기구독 후보"를 각각 최소 1개 이상 표시. 근거 부족 시 후보/추정 문구.
- **비용 상한:** CSV 1건 분석에서 LLM 입력은 매핑 샘플 ≤20행 + 집계/최소 필드만, LLM 호출은 기본 경로 3회 이하(mapColumns, insights, subscription candidates).
- **전환 관찰:** 게스트→가입 CTA, Free→Pro CTA, 결제 시작 클릭을 PII 없이 기록.
